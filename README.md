# 小组报数日记

这是一个可部署的网页版本，用于每日报数、早中晚打卡、达标审核和小组汇总统计。浏览器端不使用 `localStorage`、IndexedDB 或本地自动备份；解锁后所有记录和配置都通过云端接口写入服务端 JSON 数据文件。

## 启动

```bash
npm start
```

默认访问地址：

```text
http://localhost:8787
```

可选环境变量：

```bash
PORT=8787
APP_PASSWORD=999
DATA_FILE=./data/report_data.json
```

Windows 可以双击 `run_web_app.bat` 启动。

## 密码保护

打开页面后会先显示云端访问密码界面，默认密码是 `999`。输入正确密码后，前端才会请求云端数据并进入页面。

管理员可以在“管理员 > 权限”里修改密码并保存配置。新密码会写入云端数据；当前浏览器只在内存里保留本次会话密码，不写入本地缓存。

## 早中晚打卡

“今日”页面新增“早中晚打卡”区域，每位成员每天可以分别选择：

- 早打卡
- 中打卡
- 晚打卡

默认选项是“准时上线”和“迟到”。管理员可以在“管理员 > 打卡选项”里新增、重命名或删除打卡选项。

## 整体预览

“整体预览”里新增“各组打卡情况”，按分组汇总：

- 每组总人数
- 三个时段各选项人数
- 未打卡人数
- 全部完成打卡人数
- 出现迟到的人数

成员达标表和项目明细仍然保留，并会展示成员当天的早中晚打卡结果。

## 管理员

管理员页面可以维护：

- 默认定额、当天定额、成员个人定额
- 报数项目和换算系数
- 成员、分组、成员顺序
- 每个分组需要填写的项目
- 早中晚打卡选项
- 页面访问密码
- 自动审核和审核反馈文案

## 云端存储说明

当前内置的 `server.cjs` 是无第三方依赖的 Node.js 服务端：

- `GET /api/data` 读取云端数据
- `PUT /api/data` 写入云端数据
- `POST /api/unlock` 校验密码并返回数据

服务端数据默认写入 `data/report_data.json`。部署到云服务器、内网服务器、NAS 或支持 Node.js 的平台后，这个文件就是云端数据源。浏览器刷新或关闭后不会保留本地副本。

生产部署建议放在 HTTPS 后面，并限制服务器文件权限，避免直接暴露 `data/report_data.json`。
